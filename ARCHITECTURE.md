# 架構設計文檔

## 系統概述

智能混合輸入法是一個創新的輸入系統，能夠自動檢測使用者想要輸入中文還是英文，無需手動切換輸入模式。

## 核心組件

### 1. 資料結構層 (DataStructures/)

#### 1.1 Trie 樹 (Trie.swift)
- **用途**: 快速詞彙查找和前綴匹配
- **時間複雜度**: 
  - 插入: O(m)，m 為詞長度
  - 查找: O(m)
  - 前綴匹配: O(m + k)，k 為結果數量
- **空間複雜度**: O(ALPHABET_SIZE × N × M)
- **優化**:
  - 使用字典而非固定大小陣列節省空間
  - 支持頻率權重排序
  - 批量插入優化

#### 1.2 N-gram 語言模型 (NgramModel.swift)
- **用途**: 計算詞序列機率，支持上下文預測
- **模型類型**:
  - Unigram: P(w) - 單詞機率
  - Bigram: P(w₂|w₁) - 基於前一個詞的條件機率
  - Trigram: P(w₃|w₁,w₂) - 基於前兩個詞的條件機率
- **平滑技術**: Laplace Smoothing (加一平滑)
- **公式**:
  ```
  P(w|context) = (count(context, w) + α) / (count(context) + α × V)
  其中 α 為平滑因子，V 為詞彙表大小
  ```

### 2. 核心引擎層 (Core/)

#### 2.1 輸入引擎 (InputEngine.swift)
- **職責**: 整合所有組件，提供統一的輸入處理介面
- **主要功能**:
  - 輸入緩衝管理
  - 候選詞生成和排序
  - 上下文維護
  - 使用者選擇處理
- **工作流程**:
  ```
  使用者輸入 → 語言檢測 → 候選詞生成 → 排序 → 返回候選列表
  ```

#### 2.2 注音轉換器 (BopomofoConverter.swift)
- **職責**: 將注音符號轉換為中文字
- **演算法**:
  - **Viterbi 演算法**: 找出最佳轉換路徑
  - **動態規劃**: 處理注音分段
- **Viterbi 演算法流程**:
  ```
  1. 初始化: 設定第一個字的所有可能狀態
  2. 遞迴: 對每個位置，計算所有狀態的最大機率路徑
  3. 終止: 從最後位置回溯，得到最佳路徑
  ```
- **狀態轉移機率**: 使用 N-gram 模型計算

#### 2.3 語言檢測器 (LanguageDetector.swift)
- **職責**: 判斷輸入是中文還是英文
- **特徵提取**:
  1. 注音符號比例
  2. ASCII 字母比例
  3. 中文字符比例
  4. 數字比例
  5. 英文單詞匹配度
  6. 上下文語言
  7. 使用者歷史偏好
  8. 字串長度
- **檢測策略**:
  - 基於規則的分類
  - 加權評分機制
  - 上下文增強

### 3. 機器學習層 (ML/)

#### 3.1 語言分類器 (LanguageClassifier.swift)
- **模型類型**: 樸素貝葉斯分類器
- **數學原理**:
  ```
  P(Language|Features) ∝ P(Language) × ∏P(Feature_i|Language)
  ```
- **特徵工程**:
  - 布林特徵（是否包含特定字符）
  - 數值特徵（字符比例、平均長度）
- **訓練方式**:
  - 批量訓練
  - 增量學習（在線學習）
- **優點**:
  - 計算快速
  - 可解釋性強
  - 支持增量更新

## 演算法詳解

### Viterbi 演算法實現

```
輸入: 注音序列 [b₁, b₂, ..., bₙ]
輸出: 最佳中文序列 [c₁, c₂, ..., cₙ]

步驟:
1. 初始化 DP 表
   dp[0][c] = P(c|b₁) 對所有可能的 c

2. 遞迴計算
   for i = 1 to n:
       for each 候選字 c:
           dp[i][c] = max(dp[i-1][c'] × P(c|c') × P(c|bᵢ))
           backpointer[i][c] = argmax(...)

3. 回溯
   從 dp[n] 的最大值開始，沿著 backpointer 回溯
```

### Trie 樹結構

```
          root
         /  |  \
        a   b   c
       /    |    \
      p     a     a
     / \    |     |
    p   ple  t    r
```

每個節點存儲:
- children: 子節點映射
- isEndOfWord: 是否為詞的結尾
- frequency: 詞頻
- word: 完整詞（僅在終止節點）

## 性能考量

### 時間複雜度分析

| 操作 | 複雜度 | 說明 |
|------|--------|------|
| Trie 插入 | O(m) | m 為詞長度 |
| Trie 查找 | O(m) | |
| 前綴匹配 | O(m + k) | k 為結果數量 |
| N-gram 查詢 | O(1) | 使用哈希表 |
| Viterbi | O(n × s²) | n 為序列長度，s 為狀態數 |
| 語言檢測 | O(n) | n 為輸入長度 |

### 空間複雜度分析

| 組件 | 複雜度 | 優化策略 |
|------|--------|----------|
| Trie | O(N × M × C) | 使用字典代替陣列 |
| N-gram | O(V² 或 V³) | 只存儲出現過的組合 |
| 語言檢測 | O(H) | H 為歷史記錄大小 |

### 優化策略

1. **剪枝**
   - Viterbi 演算法中只保留 top-k 狀態
   - 候選詞限制數量

2. **緩存**
   - 常用詞彙的轉換結果
   - N-gram 機率計算結果

3. **延遲加載**
   - 詞典按需加載
   - 大型模型分塊載入

4. **並行處理**
   - 候選詞生成可並行
   - 批量輸入處理
